# 🏦 CodeAlpha Advanced Banking Suite

Welcome to the **CodeAlpha Advanced Banking Suite**, a high-fidelity, command-line mainframe application engineered to simulate a modern financial banking system. Developed by **Shaurya Tiwari**, this software leverages advanced C++ design patterns, true-color ANSI escape structures, typing animations, and robust file handling for persistent data storage.

---

## 🎨 Visual Identity & Styling

This interface is designed to break away from traditional black-and-white console apps by integrating premium neon layouts and smooth gradient typography:
- **Gradient Titles**: Banners are generated by interpolating RGB parameters character-by-character to display smooth transitions from neon magenta to neon cyan.
- **Neon Accents**: Color codes identify active segments—cyan for standard prompts, pink/magenta for errors or exit sequences, green for approved transactions, and purple for administrative options.
- **UI Windows**: Double-line borders (`╔`, `╗`, `╚`, `╝`) are dynamically calculated to fit transaction records and account details into clean, aligned panels.
- **Loading Bars**: Boot sequences and validations feature animated progress bars with smooth-filling block indicators (`█`, `▒`, `░`).

---

## 🔐 Cryptography & Mainframe Security

The banking suite integrates native security protocols to prevent unauthorized file reads and transaction tempering:
- **SHA-256 Hashing**: Includes a header-only implementation of the SHA-256 algorithm. Passwords and transaction PINs are hashed before writing them to the database.
- **Holographic Masking**: Custom POSIX terminal modifications are applied using `termios.h` to disable terminal character echoing. Sign-in inputs and PIN entry digits are masked with `*` as they are typed.
- **Input Sanitization**: Email formats, telephone digits, and currency bounds are validated at the entry level to prevent database field corruptions.

## ⚙️ Core Application Modules

### 👑 Administrative Operations
- **User Provisioning**: Admin portals allow registration of customer profiles and sequential generation of client database IDs.
- **Account Issuance**: Provision Savings accounts (with compound interest yields), Checking accounts, or Fixed Deposit products.
- **Global Accounts freeze**: Freeze/unfreeze customer accounts to halt or resume operational access.
- **Interest Dispersal**: Trigger monthly interest payouts (4.0% p.a.) globally to all active Savings accounts.
- **Audit Logs**: Access the global transaction database in a formatted tabular structure.

### 👥 Customer Services
- **Cash Counter**: Make cash deposits directly to active account numbers.
- **ATM Dispenser**: Withdraw funds securely using masked 4-digit PIN authorizations.
- **Intra-Bank Wire**: Wire funds directly to other customer accounts.
- **Ledger Inquiries**: Review personal transactions with color-coded tags (green for incoming, pink for outgoing, yellow for transfers).
- **Calculators Submenu**: Access compound interest calculators and DTI loan eligibility estimators.

## 🚀 Building & Launching

### System Requirements
- A macOS/Linux terminal environment supporting ANSI/UTF-8 color mappings.
- A C++17 compatible compiler (`clang++` or `g++`).

### Compile Steps
Open your command terminal inside the project directory and build:
```bash
clang++ -std=c++17 main.cpp -o banking_system
```

### Run Executable
```bash
./banking_system
```
